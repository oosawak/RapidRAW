# 第16章: 描画経路の分離

## 16.1 方針

- 自分が描く入力処理は WASM に寄せる
- 相手から届く描画結果は Canvas で表示する
- 共有するのは描画データと状態だけにする

## 16.2 自分の描画経路

自分のストロークはローカルで即座に反映するため、WASM で補間や平滑化を行う。
ここでは次を担当する。

- ブラシ補間
- ストロークの確定
- Undo / Redo
- ローカルのプレビュー反映

## 16.3 相手の描画経路

相手のストロークはネットワーク経由で受け取るため、描画結果を Canvas で再生する。
ここでは次を担当する。

- 受信したストロークの再生
- タイムスタンプ順の描画
- 表示専用の合成

## 16.4 共有データ

両者で共通に使うデータは、WASM 固有の内部状態ではなく、次のような中立的な形式にする。

- `stroke_id`
- `user_id`
- `tool`
- `color`
- `size`
- `points`
- `timestamp`

## 16.5 実装メリット

- 自分の描画は高速に保ちやすい
- 他人の表示は実装が単純になる
- 将来 WASM と Canvas の役割を再調整しやすい
